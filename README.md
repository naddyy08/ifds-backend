🤖 IFDS – AI-Powered Inventory Fraud Detection System for Restaurant

IFDS is an intelligent inventory management system designed to detect fraud and anomalies in restaurant stock operations.
The backend is developed using a secure and scalable architecture that integrates AI-based detection with real-time inventory tracking.

The system provides:
Real-time inventory and transaction monitoring
AI-powered anomaly detection for fraud identification
Secure role-based access control (Admin, Manager, Staff)
RESTful API for frontend integration
Automated reporting and alert system

🔗 Live API Base URL
https://ifds-backend.onrender.com/

🧰 Backend & System Architecture
🔙 Backend Development
Python (Flask Framework) – core backend API development
RESTful API – handles communication between frontend and backend
MySQL Database – structured storage for inventory, users, and transactions
Role-Based Access Control (RBAC) – secure user permission management
Secure authentication system – protects user data and access

🧠 AI & Fraud Detection Module
Machine Learning-based anomaly detection
Analyses inventory and transaction patterns
Detects unusual stock movements and suspicious activities
Helps identify potential fraud such as stock manipulation or theft

🔄 Core Backend Functional Modules
Inventory Management Module – tracks stock levels and updates
Transaction Management Module – records stock in/out activities
Fraud Detection Module – flags anomalies in real time
API Layer – connects backend services with React frontend

🔐 Security Features
Secure login and authentication system
Role-based authorization (Admin, Manager, Staff)
Protection against unauthorized access
Data validation and secure API handling

⚙️ System Development Process
The backend system was developed following structured phases:

Requirement Phase: Identified system problems such as inaccurate inventory reporting and fraud risks
Design Phase: Designed system architecture, database (ERD), and API structure
Development Phase:
Backend development using Flask API
Implementation of RBAC and security features
Integration of AI anomaly detection module
Testing Phase: Unit testing, integration testing, and system testing
Deployment Phase: Backend deployed with database configuration
Review Phase: User acceptance testing and performance improvement

🧑‍💻 About the Project
This project was developed as part of a Final Year Project (FYP) for an Information Technology program.

The system focuses on solving key issues in restaurant inventory management, especially:

False or inaccurate reporting
Inventory fraud and manipulation
Lack of real-time monitoring

By integrating AI into the backend, IFDS enhances accuracy, improves decision-making, and reduces financial losses.

This project is developed for educational and research purposes only. It demonstrates the implementation of AI and secure backend systems in detecting fraud within restaurant inventory management.
